# F-T-006 需求文档编辑器 — 手动测试流程

> **启动方式**: 在 VS Code 终端运行 `bash start_req_editor.sh`  
> **测试地址**: http://localhost:8000/req-editor

---

## 前置准备

- [ ] 终端运行 `bash start_req_editor.sh`，确认无报错
- [ ] 浏览器打开 `http://localhost:8000/req-editor`
- [ ] 页面正常显示，包含：标题栏、工具栏、左右分栏（编辑器+预览）、底部评论区域

---

## 测试项 1: 加载预置模板

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1.1 | 点击工具栏「**加载模板**」按钮 | 左侧编辑器填充 Markdown 内容 |
| 1.2 | 查看左侧编辑器内容 | 包含 `# 功能需求文档`、`## F-XXX`、`验收标准`、`安全属性` 等章节 |
| 1.3 | 查看右侧预览面板 | 实时显示渲染后的 HTML（标题、列表、表格等格式正确） |
| 1.4 | 查看底部消息条 | 显示绿色背景提示「已加载预置模板」 |

---

## 测试项 2: Markdown 实时编辑与预览

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 2.1 | 在左侧编辑器修改文字（如改 `F-XXX` 为 `F-001`） | 右侧预览**实时同步更新** |
| 2.2 | 输入 Markdown 语法 `**粗体**` | 右侧渲染为粗体文字 |
| 2.3 | 输入代码块 ` ```python\nprint('hi')\n``` ` | 右侧渲染为带高亮的代码块 |
| 2.4 | 输入表格语法 | 右侧渲染为 HTML 表格 |

## 测试项 3: Copilot 自动补全辅助

|------|------|---------|

## 测试项 4: 格式校验

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 4.1 | 清空编辑器内容（手动删除全部文字） | — |
| 4.2 | 点击「**校验格式**」按钮 | 编辑器下方显示红色错误列表：<br>• 缺少 F-xxx 功能编号<br>• 缺少「验收标准」<br>• 缺少「安全属性」 |
| 4.3 | 点击「加载模板」恢复完整内容 | 编辑器恢复到模板内容 |
| 4.4 | 再次点击「**校验格式**」 | 错误列表消失（校验通过） |

## 异常场景

| 场景 | 操作 | 预期 |
|------|------|------|
| 导入非 UTF-8 文件 | 用记事本创建 ANSI 编码文件导入 | 提示「文件编码错误」 |
| 空内容校验 | 清空后点校验 | 列出全部 3 项错误 |
| 空评论提交 | 不输入文字直接点发表 | 不添加评论 |
| 封存后点导入 | 封存状态下点导入 | 禁用状态 / 提示已封存 |
