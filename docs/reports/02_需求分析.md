## 2. 功能需求列表（F-D-009）

### 2.1 Commitlint 配置向导

| 字段 | 内容 |
|------|------|
| **需求编号** | F-D-009-01 |
| **描述** | 平台提供 commitlint 配置向导，支持选择规则集（Conventional Commits 标准或自定义）并在线编辑规则参数 |
| **优先级** | P2 |
| **输入** | rule_type（conventional/custom）、type_enum（列表）、header_max_length（默认100）、subject_min_length（默认5） |
| **输出** | 配置写入 MySQL commitlint_configs 表 |
| **验收标准** | 1. 可选择 Conventional Commits 或自定义规则集；2. type 枚举可增删编辑；3. header 最大长度和 subject 最小长度可填写；4. 保存后配置持久化 |

### 2.2 配置文件生成

| 字段 | 内容 |
|------|------|
| **需求编号** | F-D-009-02 |
| **描述** | 一键生成 .github/workflows/commitlint.yml 和 .commitlintrc.json |
| **优先级** | P2 |
| **输入** | MySQL 中已保存的配置 |
| **输出** | 两个配置文件的内容（JSON/YAML 字符串） |
| **验收标准** | 1. commitlintrc.json 包含 type-enum、header-max-length、subject-min-length 规则；2. commitlint.yml 在 pull_request 事件触发；3. 启用时规则级别为 error（2），停用时为 0 |

### 2.3 提交至仓库

| 字段 | 内容 |
|------|------|
| **需求编号** | F-D-009-03 |
| **描述** | 将生成的配置文件通过 GitHub Contents API 提交至仓库指定分支，纳入 Git 版本控制 |
| **优先级** | P2 |
| **输入** | 配置文件内容 + 目标分支 |
| **输出** | GitHub commit SHA 和 commit URL |
| **验收标准** | 1. 文件写入仓库指定分支；2. commit 信息包含版本号和启用状态；3. 返回可访问的 commit 链接 |

### 2.4 CI 联动

| 字段 | 内容 |
|------|------|
| **需求编号** | F-D-009-04 |
| **描述** | 规则生效后，PR 中任何不符合规范的 commit 消息将导致 CI 失败，对应 PR 无法合并 |
| **优先级** | P2 |
| **验收标准** | 1. CI 工作流在 pull_request 事件触发；2. 不合规 commit 导致 CI 运行失败；3. CI 失败后 PR 无法合并（需配合 F-D-002 分支保护） |

### 2.5 状态展示与失败记录

| 字段 | 内容 |
|------|------|
| **需求编号** | F-D-009-05 |
| **描述** | 平台展示当前仓库 commitlint 配置的启用/停用状态、规则版本，以及最近 5 次校验失败记录摘要 |
| **优先级** | P2 |
| **验收标准** | 1. 页面显示启用/停用状态（带颜色标识）；2. 显示规则版本号；3. 显示最近最多 5 条校验失败记录（commit hash、消息、时间） |

## 3. 非功能需求

### 3.1 响应时间

| 场景 | 目标 |
|------|------|
| 配置保存接口 | ≤ 500ms |
| 配置文件生成 | ≤ 200ms |
| GitHub 推送接口 | ≤ 5s（依赖网络） |

### 3.2 数据持久化

| 数据类型 | 存储方式 | 持久性 |
|---------|---------|--------|
| 规则配置 | MySQL commitlint_configs 表 | 永久保存 |
| 校验失败记录 | 本地 JSON 文件 | 服务重启后清空（后续改进） |

### 3.3 安全要求

| 要求 | 说明 |
|------|------|
| GitHub Token | 通过环境变量配置，不硬编码，不提交至 Git |
| 认证 | 所有 commitlint API 需要教师 JWT Token 认证 |
| SQL 注入防护 | 参数化查询 |

## 4. 限制与约束

| 约束类型 | 具体内容 |
|---------|---------|
| **操作权限** | GitHub Token 需要有仓库 Contents 写入权限 |
| **仓库限制** | 当前目标仓库固定为 uwislab/robotics-systems-course |
| **分支选择** | 通过下拉框从 GitHub API 获取分支列表 |

## 5. 数据字典（F-D-009 相关）

| 实体 | 属性 | 类型 | 说明 |
|------|------|------|------|
| commitlint_configs | rule_set | ENUM('conventional','custom') | 规则集类型 |
| commitlint_configs | header_max_len | INT | Header 最大长度 |
| commitlint_configs | subject_min_len | INT | Subject 最小长度 |
| commitlint_configs | type_enum_json | JSON | 允许的 type 枚举列表 |
| commitlint_configs | enabled | TINYINT(1) | 启用状态 |
| commitlint_configs | config_version | INT | 规则版本号（自增） |
