## 2. 表结构说明

### commitlint_configs（Commitlint 配置表）

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | BIGINT | PRIMARY KEY AUTO_INCREMENT | 自增主键 |
| course_id | BIGINT | NOT NULL, UNIQUE | 所属课程 ID（关联 courses） |
| rule_set | ENUM('conventional','custom') | DEFAULT 'conventional' | 规则集类型 |
| header_max_len | INT | NOT NULL DEFAULT 100 | Header 最大长度 |
| subject_min_len | INT | NOT NULL DEFAULT 5 | Subject 最小长度 |
| type_enum_json | JSON | DEFAULT NULL | 允许的 type 枚举（JSON 数组） |
| enabled | TINYINT(1) | NOT NULL DEFAULT 0 | 是否启用（1=启用，0=停用） |
| config_version | INT | NOT NULL DEFAULT 1 | 规则版本号（每次更新自增） |
| updated_by | BIGINT | NOT NULL | 更新人用户 ID（关联 teachers） |
| updated_at | DATETIME | DEFAULT CURRENT_TIMESTAMP ON UPDATE | 记录更新时间 |

### 关联表

| 表名 | 关联字段 | 说明 |
|------|---------|------|
| courses | id | commitlint_configs.course_id → courses.id |
| teachers | user_id | commitlint_configs.updated_by → teachers.user_id |

---

## 3. 索引策略

| 表名 | 索引字段 | 索引类型 | 理由 |
|------|---------|---------|------|
| commitlint_configs | course_id | UNIQUE | 每课程一条配置，按 course_id 精确查询 |
| commitlint_configs | updated_by | INDEX | 按更新人查询历史配置 |

---

## 4. 关联关系

| 子表 | 外键字段 | 父表 | 参照字段 | 级联行为 |
|------|---------|------|---------|---------|
| commitlint_configs | course_id | courses | id | ON DELETE RESTRICT |
| commitlint_configs | updated_by | teachers | user_id | ON DELETE RESTRICT |

> RESTRICT 策略：存在关联配置时，不允许删除课程或教师记录。

## 5. 数据初始化脚本

```sql
-- 创建默认配置（首次使用时自动执行）
SET foreign_key_checks=0;

INSERT IGNORE INTO courses (id) VALUES (1);
INSERT IGNORE INTO teachers (user_id) VALUES (1);

INSERT INTO commitlint_configs
  (course_id, rule_set, header_max_len, subject_min_len,
   type_enum_json, enabled, config_version, updated_by)
VALUES
  (1, 'conventional', 100, 5,
   '["feat","fix","refactor","style","test","docs","chore"]',
   1, 1, 1)
ON DUPLICATE KEY UPDATE id=id;

SET foreign_key_checks=1;
```

### 初始化逻辑说明

项目启动时，`ensure_mysql_row()` 函数自动检查 `commitlint_configs` 表中是否存在 `course_id=1` 的记录：

- **存在** → 不做任何操作
- **不存在** → 插入默认配置（Conventional Commits 标准，7 个 type 枚举，enabled=1）

## 6. 字段映射说明

由于后端 API 使用前端友好字段名，而 MySQL 列名使用不同命名风格，建立了完整的映射机制：

| 前端字段 | MySQL 列名 | 说明 |
|---------|-----------|------|
| rule_type | rule_set | 规则集类型 |
| type_enum | type_enum_json | Type 枚举列表（JSON） |
| header_max_length | header_max_len | Header 最大长度 |
| subject_min_length | subject_min_len | Subject 最小长度 |
| rule_version | config_version | 规则版本号 |

映射在 [`commitlint.py`](file:///d:/Projects/Java/OA-EPP/backend/app/routers/commitlint.py) 中通过 `_FRONTEND_TO_MYSQL` 和 `_MYSQL_TO_FRONTEND` 两个字典实现双向转换。

## 7. Failures 存储（本地 JSON 方案）

校验失败记录因 MySQL 无对应表，采用本地 JSON 文件存储：

| 属性 | 说明 |
|------|------|
| **存储位置** | `.local_exam_data/commitlint_failures.json` |
| **最大记录数** | 5 条 |
| **数据结构** | `{id, commit_sha, commit_msg, author, branch, pr_number, failed_at, error_msg}` |
| **管理接口** | `failures_store.py` 中的 `get_failures()` / `add_failure()` |
