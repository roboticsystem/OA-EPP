## 3. 交互流程说明

### 3.1 核心流程：配置 → 推送 → 生效

```
┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐
│ 打开页面  │──▶│ 编辑规则  │──▶│ 点击保存  │──▶│ 配置持久 │
│ 自动加载  │   │ 增删 type │   │ 并推送到  │   │ 化 + 生  │
│ 当前配置  │   │ 调整参数  │   │ 仓库     │   │ 成文件 + │
└──────────┘   └──────────┘   └──────────┘   │ 推送到   │
                                              │ GitHub   │
                                              └──────────┘
                                                    │
                                              ┌─────▼──────┐
                                              │ CI 自动触发 │
                                              │ 不合规commit│
                                              │ → CI 失败   │
                                              │ → PR 无法合│
                                              └────────────┘
```

### 3.2 分支选择交互

```
用户点击下拉框
    ↓
loadBranches() 调用 GET /branches
    ↓
从 GitHub API 获取仓库所有分支
    ↓
渲染下拉选项，默认选中仓库 default_branch
    ↓
用户选择其他分支 → 后续 push/save-and-push 使用所选分支
```

## 5. 可用性自评（Nielsen 10 原则）

### 5.1 原则一：系统状态的可见性

- 加载时显示"加载中…"，按钮在提交时显示"⏳ 提交中…"并禁用
- 保存成功显示绿色 toast，失败显示红色错误
- 仓库状态区实时显示文件是否存在

### 5.2 原则三：用户控制和自由度

- Type 标签支持点击 ✕ 移除
- 提交前可预览配置文件内容确认
- 所有按钮均有清晰的文字标识

### 5.3 原则四：一致性和标准

- 颜色含义一致：绿色=成功，红色=错误，橙色=警告，灰色=不可用
- 按钮样式统一：.btn + 颜色修饰类
- 表单布局统一：.form-group + .form-row
