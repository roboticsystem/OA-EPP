## 3. 核心算法说明

### 3.1 commitlintrc 配置生成算法

```python
def build_commitlintrc(
    type_enum: list,           # 如 ["feat","fix","refactor",...]
    header_max_length: int,    # 如 100
    subject_min_length: int,   # 如 5
    is_enabled: bool = True,   # 启用状态
) -> str:
    """
    算法说明:
    1. 根据 is_enabled 确定规则级别 level
       - True  → level = 2 (error，CI 阻断)
       - False → level = 0 (关闭，不阻断)
    2. 构建 rules 字典，包含三条规则
    3. 将扩展 + rules 序列化为 JSON
    """
    level = 2 if is_enabled else 0
    rules = {
        "type-enum":           [level, "always", sorted(set(type_enum))],
        "subject-min-length":  [level, "always", subject_min_length],
        "header-max-length":   [level, "always", header_max_length],
    }
    rc = {
        "extends": ["@commitlint/config-conventional"],
        "rules": rules,
    }
    return json.dumps(rc, indent=2, ensure_ascii=False)
```

### 3.2 GitHub Contents API 推送算法

```python
def push_files(files, commit_message, branch=None):
    """
    算法说明:
    1. 确定目标分支（默认 GIT_BRANCH）
    2. 遍历每个文件:
       a. 调用 get_file_sha() 检查文件是否已存在
       b. 构建请求体: message, content(base64), branch, sha(若更新)
       c. 调用 PUT /repos/{owner}/{repo}/contents/{path}
    3. 返回最后一个 commit 的信息
    """
    branch = branch or GIT_BRANCH
    last_sha = None
    for f in files:
        data = {
            "message": commit_message,
            "content": base64.b64encode(f["content"].encode()),
            "branch": branch,
        }
        sha = get_file_sha(f["path"], branch)  # 检查文件是否存在
        if sha:
            data["sha"] = sha  # 存在则更新
        resp = _github_request("PUT", _api_url(f"/contents/{f['path']}"), data)
        last_sha = resp["commit"]["sha"]
    return {"commit_sha": last_sha, "commit_url": f"https://github.com/.../commit/{last_sha}"}
```

### 3.3 前端字段与 MySQL 列名映射算法

```python
# 双向映射字典
_FRONTEND_TO_MYSQL = {
    "rule_type": "rule_set",
    "type_enum": "type_enum_json",
    "header_max_length": "header_max_len",
    "subject_min_length": "subject_min_len",
    "rule_version": "config_version",
    "enabled": "enabled",
}
_MYSQL_TO_FRONTEND = {v: k for k, v in _FRONTEND_TO_MYSQL.items()}

def _row_to_frontend(row: dict) -> dict:
    """MySQL 行数据 → 前端字段名"""
    result = {}
    for mysql_col, value in row.items():
        frontend_col = _MYSQL_TO_FRONTEND.get(mysql_col, mysql_col)
        result[frontend_col] = value
    # JSON 字段反序列化
    if result.get("type_enum") and isinstance(result["type_enum"], str):
        result["type_enum"] = json.loads(result["type_enum"])
    return result
```

## 4. 错误处理策略

### 4.1 F-D-009 异常处理

| 异常类型 | 触发场景 | 前端提示 | HTTP 状态码 |
|---------|---------|---------|------------|
| 教师未登录 | 未携带或无效 Token | "请先登录" | 401 |
| 配置不存在 | 数据库无 course_id 对应行 | "配置不存在" | 404 |
| 参数校验失败 | header_max_length < 1 | "必须为正整数" | 422 |
| rule_type 不合法 | 非 conventional/custom | "必须为 conventional 或 custom" | 422 |
| type_enum 为空 | 空列表 | "type_enum 不能为空列表" | 422 |
| GitHub Token 无效 | Token 未配置或过期 | "连接失败请检查" | 502 |
| GitHub 推送失败 | 网络异常/权限不足 | "提交至仓库失败: {具体错误}" | 502 |

## 5. 安全设计

### 5.1 认证机制

- 所有 commitlint API 使用 `_require_teacher()` 统一校验教师 JWT Token
- Token 从 HTTP Header `Authorization: Bearer <token>` 提取
- 角色校验确保非教师角色被拒绝

### 5.2 GitHub Token 安全

- Token 通过环境变量 `GITHUB_TOKEN` 读取，不硬编码在代码中
- Token 不在前端暴露，仅在后端 `github_client.py` 中使用
- Token 有效性通过 `check_token()` API 在前端展示（不暴露 Token 值）

### 5.3 SQL 注入防护

- `mysql_db.py` 中的 `mysql_db()` 上下文管理器使用参数化查询（`%s` 占位符）
- 禁止拼接 SQL 字符串

## 6. 性能设计

### 6.1 查询优化

- `ensure_mysql_row()` 启动时一次性检查配置是否存在
- 配置读取使用主键 course_id 精确查询

### 6.2 外部 API 限流

- GitHub API 需注意速率限制（5000 次/小时）
- `list_branches()` 仅获取 100 条分支，避免分页

### 6.3 前端缓存

- 配置数据在页面加载时一次性获取
- 失败记录在每次操作后重新加载
