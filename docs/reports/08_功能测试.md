## 6. 缺陷跟踪（F-D-009 相关）

| Bug ID | 描述 | 严重程度 | 修复状态 | 解决方案 |
|--------|------|---------|---------|---------|
| BUG-001 | `from` 误写为 `rom` 导致 SyntaxError | 严重 | ✅ 已修复 | `rom` → `from` |
| BUG-002 | SQLite 数据库目录不存在 | 严重 | ✅ 已修复 | 自动创建 `.local_exam_data` 目录 |
| BUG-003 | Windows 端口被占用 WinError 10048 | 中等 | ✅ 已修复 | 改用 8013 端口 |
| BUG-004 | GitHub Git Data API 返回 403 | 严重 | ✅ 已修复 | 改用 Contents API |
| BUG-005 | MySQL 外键约束导致初始化失败 | 中等 | ✅ 已修复 | `SET foreign_key_checks=0` 绕过 |
| BUG-006 | `_build_commitlintrc` 函数删除后调用残留 | 严重 | ✅ 已修复 | 改为 `_load_config()` + `build_commitlintrc()` |

## 7. 遗留问题

| 问题 | 影响 | 原因 | 建议 |
|------|------|------|------|
| GitHub 账号对仓库 `push=False` | 无法推送文件至仓库 | 组织仓库需要管理员授权 Collaborator 写权限 | 联系教师添加 Write 权限 |
| 失败记录使用本地 JSON 文件 | 服务重启后记录丢失 | MySQL 无 `commitlint_failures` 表 | 后续迁移到 MySQL 独立表 |
| 未实现 Webhook 自动记录失败 | 失败记录需手动录入 | 不在当前需求范围内 | 后续可接入 GitHub Webhook |
