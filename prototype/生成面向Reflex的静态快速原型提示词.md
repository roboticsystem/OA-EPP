# 生成面向 Reflex 的静态快速原型——完整提示词与规范

> 本文档汇总了生成 OA-EPP（工程实践综合管理平台）静态快速原型的全部要求、页面清单、设计规范及需求覆盖对照表。
> 适用于使用 Lovable / Stitch / v0 等 AI 工具生成原型，或手工编写 HTML 原型，目标是最终能 **1:1 复现到 Reflex**。

---

## 一、快速原型的目标与必要性

快速原型有以下三个核心作用：

1. **统一全班视觉基准**：原型确立配色、布局、组件风格，保证所有同学实现出来的系统外观一致，避免各自为政。
2. **降低沟通成本**：可交互/可点击的静态原型是需求澄清的最高效载体，比文字描述或 UML 图更直观。
3. **1:1 指导 Reflex 实现**：原型中的每一个 HTML 结构、Tailwind 类名、布局关系，可以直接翻译为 Reflex 组件代码，减少设计到实现的信息损耗。

**推荐工具**：
| 工具 | 特点 | 与 Reflex 映射质量 |
|------|------|------------------|
| **Lovable**（首推） | 免费，输出结构化 HTML + Tailwind，与 Reflex 几乎 1:1 | ⭐⭐⭐⭐⭐ |
| **Stitch** | 免费，视觉一致，需额外一步结构转换 | ⭐⭐⭐⭐ |
| **v0**（Vercel） | 体验最佳，付费 | ⭐⭐⭐⭐⭐ |

## 二、生成规范（强制约束）

### 2.1 技术规范

| 规则 | 要求 | 原因 |
|------|------|------|
| 样式框架 | **只用 Tailwind CSS v3 CDN**，无任何自定义 CSS | Reflex 内置 Tailwind，直接对应 |
| 禁止 `<style>` 标签 | 不允许任何内联 `<style>` 块 | 防止与 Tailwind 冲突 |
| 禁止 JavaScript | 所有 `<script>` 标签（含 CDN 引用）均不允许 | Reflex 用 Python 事件处理，JS 逻辑无法迁移 |
| 页面跳转 | 使用 `<a href="xxx.html">` 标签 | 对应 Reflex 的 `rx.link()` 组件 |
| 图表 | 用纯 SVG 占位（矩形 + 文字说明） | Reflex 实现时替换为 `recharts`/内置图表组件 |
| 图片 | 用 `bg-gray-200` 色块占位，标注尺寸 | 无需真实图片，减少依赖 |
| 字符集 | `<meta charset="UTF-8">` + `<html lang="zh-CN">` | 确保中文正常渲染 |
| 响应式 | 使用 Tailwind `sm:`/`md:`/`lg:` 断点 | 对应 Reflex 移动端适配 |

### 2.2 视觉规范

| 属性 | 学生端 | 教师端 |
|------|-------|-------|
| 主色调 | 蓝色 `blue-700` / `blue-600` | 靛蓝 `indigo-700` / `indigo-600` |
| 背景色 | `bg-gray-50` | `bg-gray-50` |
| 卡片 | `bg-white rounded-xl border border-gray-200 shadow-sm` | 同 |
| 按钮（主） | `bg-blue-600 hover:bg-blue-700 text-white rounded-lg` | `bg-indigo-600 hover:bg-indigo-700 text-white rounded-lg` |
| 按钮（次） | `border border-gray-300 text-gray-600 rounded-lg` | 同 |
| 状态：成功 | `bg-green-100 text-green-600` | 同 |
| 状态：警告 | `bg-yellow-100 text-yellow-600` | 同 |
| 状态：错误 | `bg-red-100 text-red-600` | 同 |
| 状态：信息 | `bg-blue-100 text-blue-600` | `bg-indigo-100 text-indigo-600` |
| 字体大小 | 正文 `text-sm`，标题 `text-xl font-bold` | 同 |
| 图标 | SVG inline，`w-4 h-4` 或 `w-5 h-5`，`stroke="currentColor"` | 同 |

### 2.3 布局规范

所有页面采用**顶部导航栏 + 左侧边栏 + 右侧主内容区**三段式布局：

```
┌──────────────────────────────────────────────────────┐
│  顶部导航（bg-blue-700，高度约 48px）                  │
├──────────────┬───────────────────────────────────────┤
│  左侧边栏     │  主内容区                              │
│  w-56        │  flex-1 p-6                           │
│  bg-white    │  bg-gray-50                           │
│  border-r    │                                       │
└──────────────┴───────────────────────────────────────┘
```

- 顶部导航：左侧系统名称 + 面包屑，右侧用户名 + 头像
- 侧边栏：竖向菜单，当前页高亮（`bg-blue-50 text-blue-700`）
- 主内容：`<h1>` 页面标题 + 功能卡片区域

## 四、每个页面的详细要求

### 4.1 `login.html` — 登录页

- 居中卡片布局（`max-w-md mx-auto`）
- 系统 Logo / 名称 "OA-EPP 工程实践平台"
- 学号/邮箱输入框（`type="text"` + 占位符）
- 密码输入框（`type="password"`）
- 登录按钮（蓝色主按钮，全宽）
- 角色切换链接（学生端 / 教师端）
- 错误提示区域（红色，默认隐藏）

### 4.2 `dashboard.html` — 学生仪表盘

**核心组件**：
- 四维度得分卡片（出勤、考试、代码、PR），每个显示当前分/满分
- 雷达图 SVG 占位（四维度展示，用矩形框 + "雷达图占位" 标注）
- 柱状图 SVG 占位（历次考试得分趋势）
- 折线图 SVG 占位（全期总得分趋势）
- 任务完成率卡片（待提交数量、已提交数量、即将到期任务）
- 课程筛选下拉框（工程实践1-4）
- 课程公告区（列表，含"未读"标签，绿/橙区分）
- 待提交任务快捷入口

### 4.3 `courses.html` — 课程学习

- 工程实践1-4 四课程卡片（含：总分/章节数/已完成任务数/进度条/截止提醒/状态标签）
- 当前学期（EP4）章节明细展开表格，列含：章节号、章节名、类型（作业/考试/签到）、截止时间、状态、评分标准

### 4.4 `assignments.html` — 作业提交

- 左侧：任务分状态列表（待提交/已提交/已批改/迟交，不同颜色标签）
- 右侧提交面板：
  - 任务说明文本区域（只读）
  - 拖拽上传文件区域（虚线边框，支持格式 pdf/docx/zip/py/c/cpp）
  - 版本号信息（自动递增）
  - 提交时间戳占位
  - 提交按钮（蓝色）
- 版本历史卡片（表格：版本号/提交时间/文件名/状态/操作）
- 迟交状态黄色/橙色警告条

### 4.5 `grades.html` — 成绩与反馈

- 顶部四维度分数卡片（出勤/考试/代码/PR，含各维度满分）
- Tab 切换：代码评阅 / 考试成绩 / 出勤记录 / 学习时间线
- 代码评阅 Tab：任务列表表格（任务名、提交时间、得分、批改人、操作）+ 选中任务的教师评语卡片（含扣分项列表、改进建议、二次提交按钮）
- 成绩单 Excel 导出按钮
- 时间线 Tab：竖向时间轴（任务发布→提交→批改→反馈 四个节点，含时间戳）

### 4.6 `profile.html` — 个人资料

- 左侧：基础信息编辑表单（学号只读、姓名、班级、邮箱、联系方式）+ 保存按钮
- GitHub 绑定区块：
  - 当前绑定状态（已绑定：绿色；待审核：黄色；未绑定：灰色）
  - 输入框填写 GitHub 用户名 + "提交绑定申请"按钮
  - 待审核说明文字
- 修改密码表单（旧密码/新密码/确认密码）

### 4.7 `attendance.html` — 课堂签到

- 当前活跃签到卡片（绿色高亮边框）：
  - 闪烁绿点 + "正在进行中" 标签
  - 课程名称 + 教师 + 地点 + 时间
  - 右侧倒计时数字（秒）
  - 进度条（60 秒窗口期，绿色填充）
  - "确认签到"按钮（绿色）
  - 地理围栏提示（黄色信息条）
- 本学期出勤汇总统计（总次数/出勤/迟到/缺勤）
- 出勤率进度条（蓝色）
- 出勤记录明细表格（周次/日期/课程/签到时间/状态）
- 出勤得分计算规则说明卡片

### 4.8 `exam.html` — 在线考试

- 考试进行中提示卡片（橙色高亮边框）：
  - 考试名称 + 题目数 + 总分 + 限时
  - 右侧倒计时（分:秒）
  - 答题进度条
  - 题目序号导航（绿=已答/橙=当前/灰=未答）
- 当前题目区域：题目序号+类型+分值、题干、选项（单选带高亮已选）、上一题/下一题按钮
- 提交试卷按钮（红色，含注意事项文字）
- 历史考试成绩表格（考试名/时间/总分/得分/状态/查看详情）

### 4.9 `admin_students.html` — 教师学生管理

- 顶部四卡片统计（全班学生数/已绑定/待审核/未绑定）
- 搜索框 + 绑定状态筛选下拉 + 批量操作按钮（批量通过/发提醒）
- 学生详情表格：学号 / 姓名 / GitHub 账号 / 绑定状态 / GitHub 实名状态 / 最近核查时间 / 操作（通过/拒绝/重新核查）
  - 未绑定学生行用红色底色高亮
- 分页控件
- 右侧 GitHub 快捷链接面板（仓库、PR、Issues、Actions 等）

### 4.10 `admin_devops.html` — 开发运维

**仓库初始化向导**（步骤1~8）：
- 步骤列表（每步：序号圆圈/步骤名/状态图标/说明文字）
- 已完成步骤绿色对勾，当前步骤蓝色高亮，待完成灰色
- 脚本控制台（黑底绿字/白字 `font-mono text-sm`）：逐行显示执行输出
- 进度汇总（`Step 3/8 已完成`）

**分支保护规则**：
- PR 审查必须通过 Toggle
- 禁止 Force Push Toggle
- Issue 关闭必填 PR 编号 Toggle（F-T-007）

**AI 代码审查提示词编辑器**（F-D-007）：
- 模板选择下拉（通用/工程规范/安全审查）
- 占位符提示标签（`{diff}` `{pr_title}` `{pr_description}`）
- 多行 `<textarea>` 编辑区（`font-mono`）
- 操作按钮：保存并应用 / 验证配置 / 清除验证痕迹

**GitHub 快捷链接面板**（F-D-008）：
- 仓库主页 / Pull Requests / Issues / Actions / 分支管理 / Settings→Secrets / Settings→Branches

### 4.11 `admin_grades.html` — 成绩管理

**Tab 1：成绩单导出（F-T-008）**：
- 筛选区（课程/班级/学期三个下拉）+ 预览按钮
- 预览表格（学号/姓名/班级/出勤/考试/代码/PR/总评/等级/备注可编辑格）
- 导出按钮（Excel，文件名格式：`课程名_班级_学期_成绩单_日期.xlsx`）

**Tab 2：开发日志导出（F-T-010）**：
- 9 维度说明卡片（分支记录/提交历史/代码质量/PR情况/PR分析质量/教师评语/在线考试/考勤情况/课程得分）
- 选择学生下拉 + 导出格式选择（PDF/HTML/Excel）
- 单人导出按钮 + 批量导出全班 ZIP 按钮
- 数据说明文字（实时从数据库+GitHub API 聚合）

### 4.12 `admin_settings.html` — 系统设置

**Tab 1：学生名单导入（F-T-005）**：
- 文件拖拽上传区域（虚线边框，只支持 `.csv`，注明必填列）
- 导入模式单选（增量/全量覆盖）
- 校验结果预览表格（正常行绿色，错误行红色底色 + 错误说明）
- 确认导入/重新上传按钮 + 查看导入日志链接

**Tab 2：仓库权限配置（F-T-009）**：

- Team 配置（Team 名称/权限级别 下拉）
- 邀请状态看板（已接受/待接受/未绑定 统计 + 进度条 + 表格：学号/姓名/GitHub账号/邀请状态/操作）
- 批量邀请全班按钮 + 撤销全班权限按钮（红色边框）

**Tab 3：VS Code 扩展配置（F-T-002）**：

**Tab 4：需求文档编辑器（F-T-006）**：
- 工具栏（导入.md / 预览渲染 / 审阅模式）
- 模板插入快捷按钮（功能需求块/验收标准块/安全属性块）
- Markdown 编辑区（`font-mono`，显示文件名标签 + 格式合规状态）
- 操作按钮：提交到 GitHub / Copilot 补全建议 / 格式校验
- 文档封存状态（已封存=学生只读 / 解封编辑）

## 七、从原型到 Reflex 的对应关系

| HTML 结构 | Reflex 组件 | 说明 |
|-----------|------------|------|
| `<div class="flex gap-4">` | `rx.hstack(gap="4")` | 水平排列 |
| `<div class="flex flex-col">` | `rx.vstack()` | 垂直排列 |
| `<button class="bg-blue-600 ...">` | `rx.button(..., color_scheme="blue")` | 按钮 |
| `<input type="text" class="border ...">` | `rx.input(placeholder="...")` | 输入框 |
| `<select class="...">` | `rx.select([...])` | 下拉选择 |
| `<textarea class="...">` | `rx.textarea()` | 多行文本 |
| `<a href="page.html">` | `rx.link("文字", href="/page")` | 链接 |
| `<table>` 结构 | `rx.table.root(rx.table.row(...))` | 表格 |
| SVG 图表占位 | `rx.recharts.radar_chart(...)` | 雷达图 |
| SVG 图表占位 | `rx.recharts.bar_chart(...)` | 柱状图 |
| SVG 图表占位 | `rx.recharts.line_chart(...)` | 折线图 |
| 状态标签色块 | `rx.badge(..., color_scheme="green")` | 徽标 |
| 进度条 | `rx.progress(value=75)` | 进度条 |
| Toggle 开关 | `rx.switch(checked=State.enabled, on_change=State.toggle)` | 开关 |
